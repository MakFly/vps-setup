# Getting Started

VPS Setup is a modern CLI tool for provisioning VPS servers using Ansible. It lets you manage multiple servers, reusable configuration profiles, and a full deployment history â€” all from an elegant, interactive command-line interface.

## Prerequisites

Before installing VPS Setup, make sure the following tools are available on your machine:

| Tool | Version | Install |
|------|---------|---------|
| **Ansible** | >= 2.14 | `pip install ansible` or `brew install ansible` |
| **SSH** | â€” | Must have SSH access to your target servers |

## Installation

### Via install script (recommended)

The fastest way to get started:

```bash
curl -fsSL https://raw.githubusercontent.com/MakFly/vps-setup/main/install.sh | bash
```

### Direct download

Download the pre-compiled binary for your platform:

**Linux x64**
```bash
curl -fsSL https://github.com/MakFly/vps-setup/releases/latest/download/vps-setup-linux-x64 \
  -o ~/.local/bin/vps-setup && chmod +x ~/.local/bin/vps-setup
```

**Linux ARM64**
```bash
curl -fsSL https://github.com/MakFly/vps-setup/releases/latest/download/vps-setup-linux-arm64 \
  -o ~/.local/bin/vps-setup && chmod +x ~/.local/bin/vps-setup
```

**macOS (Apple Silicon)**
```bash
curl -fsSL https://github.com/MakFly/vps-setup/releases/latest/download/vps-setup-darwin-arm64 \
  -o ~/.local/bin/vps-setup && chmod +x ~/.local/bin/vps-setup
```

**macOS (Intel)**
```bash
curl -fsSL https://github.com/MakFly/vps-setup/releases/latest/download/vps-setup-darwin-x64 \
  -o ~/.local/bin/vps-setup && chmod +x ~/.local/bin/vps-setup
```

**Windows x64**
```powershell
Invoke-WebRequest -Uri "https://github.com/MakFly/vps-setup/releases/latest/download/vps-setup-windows-x64.exe" -OutFile "vps-setup.exe"
```

### From source

Build VPS Setup from source using [Bun](https://bun.sh):

```bash
git clone https://github.com/MakFly/vps-setup.git
cd vps-setup
bun install
bun run build
cp dist/vps-setup ~/.local/bin/
```

## First Server Setup

### 1. Initialize configuration

Run `init` to create the configuration directory and default profiles:

```bash
vps-setup init
```

This creates the `~/.config/vps-setup/` directory with a `config.yml` and three default profiles: `full-stack`, `minimal`, and `security-only`.

### 2. Add your first server

```bash
vps-setup server add prod-web --host 192.168.1.100 --user root
```

You can also specify port, tags, and SSH key:

```bash
vps-setup server add prod-db \
  --host 10.0.0.50 \
  --user admin \
  --port 2222 \
  --tags database,production
```

### 3. Test the SSH connection

Before provisioning, verify that VPS Setup can reach your server:

```bash
vps-setup server test prod-web
```

A successful test will display the detected OS and version.

### 4. Provision with a profile

```bash
vps-setup setup prod-web --profile full-stack
```

The CLI will:
1. Verify the SSH connection
2. Show which components will be installed
3. Ask for confirmation
4. Run the Ansible playbook

You can also use `--dry-run` to simulate the provisioning without making any changes:

```bash
vps-setup setup prod-web --profile full-stack --dry-run
```
